# User flows / навигация

## Роли

На `/` пользователь выбирает роль:
- **Пользователь** — бронирует столики и управляет своими бронями.
- **Админ** — управляет бронями, схемой зала и смотрит аналитику.

Роль сохраняется в `localStorage` (`rb_app_role`) и используется для guard’а роутов админа.

## Пользовательский сценарий (user)

1. `/` → выбор роли “Пользователь”
2. вкладка “Забронировать”:
   - выбор ресторана
   - выбор даты/времени/гостей
   - выбор столика
   - подтверждение бронирования
3. вкладка “Мои брони”:
   - список бронирований
   - отмена брони с подтверждающим диалогом

## Админский сценарий (admin)

1. `/` → выбор роли “Администратор”
2. `/admin` → редирект на `/admin/bookings`
3. `/admin/bookings`:
   - просмотр списка бронирований
   - поиск/фильтрация
   - смена статуса (подтвердить/отменить/завершить)
   - создание брони вручную (модалка)
4. `/admin/floor`:
   - редактор схемы зала
   - drag & drop столиков между категориями
   - сохранение схемы (persist в localStorage)
5. `/admin/analytics`:
   - live-метрики
   - список live событий
   - графики и популярность зон (моки, обновление каждые 10 секунд)
